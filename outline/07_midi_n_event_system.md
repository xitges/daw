## MIDI / Event System

외부 MIDI 장치 입력을 실시간으로 분류해 악기 채널 또는 클립 트리거로 라우팅할 수 있도록 설계.

### 입력 흐름

```
MIDI 하드웨어
  → MIDI 스레드 콜백
  → 메시지 큐에 적재
  → 오디오 콜백 내에서 pump (오디오 스레드)
  → 노트 분류
      ├─ Instrument  → 해당 채널 악기 버퍼에 추가
      └─ ClipTrigger → lock-free FIFO에 enqueue
  → CC → lock-free FIFO에 enqueue
```

MIDI 스레드에서 노트를 즉시 처리하지 않고 오디오 콜백 내에서 처리하는 이유는 샘플 단위 타이밍 정확도를 보장하기 위해서. 
오디오 콜백이 실행될 때 큐에 쌓인 메시지를 블록 내 도착 시각을 기준으로 sample offset과 함께 꺼낸다. 이렇게 하면 블록의 몇 번째 샘플에서 노트가 시작되어야 하는가를 알 수 있어, 두 콜백 사이 언젠가 도착했다는 사실만 아는 것보다 정밀한 타이밍으로 소리를 발생시킬 수 있음.

### 라우팅 분류

MIDI 라우터가 노트별 라우팅을 결정. 모든 설정은 메시지 스레드에서 atomic으로 기록하고, 오디오 스레드에서 읽는다.

```
분류 기준 (우선순위 순):
  패드 범위 내 노트        → ClipTrigger (런치패드 트리거)
  per-note 명시 설정       → 지정된 DAW 채널로 Instrument 라우팅
  나머지                   → 기본 악기 채널 (단일 채널 동작과 호환)
```

설정이 없으면 모든 노트가 기본 채널로 흘러 기존 단일 채널 동작과 완전히 호환된다.

### 메시지 스레드 드레인

오디오 스레드에서 enqueue된 이벤트는 메시지 스레드 타이머에서 꺼내기.

```
타이머
  ├─ ClipTrigger FIFO drain → 클립 런처에 전달 → 런치패드 클립 트리거
  └─ CC FIFO drain          → CC 번호별 파라미터 매핑 적용
       CC 70–77 → 채널 0–7 볼륨
       CC 10    → 채널 pan
```

오디오 스레드에서는 할당 없이 push만 하고, 메시지 스레드에서 drain하는 단방향 구조. lock-free FIFO를 사용해 두 스레드 간 어떤 락도 없이 데이터를 전달한다.
