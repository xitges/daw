# File I/O & management

---
프로젝트 전체를 XML 포맷으로 직렬화해 저장하고 복원할 수 있도록 설계.

### 직렬화 구조

직렬화 모듈이 프로젝트 구조체를 `.studioproj` XML로 읽고 쓰도록. 루트 컴포넌트가 파일 명령 네 가지(New / Open / Save / Save As)를 소유하고 직렬화 모듈에 위임하는 방식.

파일 선택 다이얼로그는 비동기로 동작, 콜백이 완료되기 전에 선택기 객체가 해제되지 않도록 공유 소유권으로 수명 관리.

### Save 흐름

```
1. 현재 편집 내용을 활성 패턴에 flush
2. 플러그인 내부 상태를 수집해 base64로 인코딩
3. 전체 프로젝트 구조를 XML로 직렬화하여 파일에 기록
4. dirty flag 해제
```

저장 경로가 미설정 상태이면 Save As로 자동 분기한다.

### Open / New 흐름

```
Open: XML 역직렬화 → 프로젝트 구조체 교체 → UI 전체 재로드
New:  프로젝트 구조체 초기화              → UI 전체 재로드
```

UI 재로드는 오디오 엔진을 먼저 정지시킨 뒤, 샘플 버퍼·오디오 클립·런치패드 샘플을 순서대로 다시 로드. 오디오 스레드가 구 데이터를 읽는 도중 교체가 일어나는 상황을 막기 위해서다. undo 히스토리도 이 시점에 전부 초기화. 새 프로젝트와 무관한 이전 액션이 스택에 남아 있으면 해제된 데이터를 참조하게 되기 때문.

### 직렬화 원칙

- **Sparse 저장**: 기본값과 동일한 필드는 XML에 생략해 파일 크기를 줄인다.
- **하위 호환**: 모든 필드 읽기에 기본값을 명시해 구버전 파일에서 누락된 필드가 생기지 않도록 한다. 구 포맷(단일 variation, 구 채널 구조 등)은 로드 시점에 자동 마이그레이션.
